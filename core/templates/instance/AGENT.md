# AGENT.md —— 本数字员工的生效组合契约

> 这是所有记忆文件如何协同生效的唯一说明书，由 liliclaw 框架维护、随运行提示词同步。
> **无论哪个内核（Claude Code / Kimi Code / Codex / Gemini / Qwen…）、无论是调度网关自动拉起还是有人手动打开本 workspace，都按本契约工作。**
> 记忆文件全部在 `.liliclaw/` 目录下，与 workspace 里用户自己的项目文件严格隔离、互不干扰。

## 文件组合（每个文件一个职责，不重叠）

| 文件 | 职责 | 何时加载 | 何时更新 |
|---|---|---|---|
| SOUL.md | 人格：我是谁、工作观、说话方式 | 每次运行开头（reflect 除外） | 仅老板调教时 |
| CONSTITUTION.md | 规则：铁律、决策边界、行为红线 | 触到边界时必读；复盘全读 | 仅老板授权时 |
| STATE.md | 目标树、现状、可用资源 | 每次运行开头 | 现状变化时 |
| TASKS.md | 任务队列 + 时间表（at=激活时间） | 每次运行开头 | 每次运行结束必更新 |
| EXPERIENCE.md | 教训（活跃规则）、打法、进化台账 | 活跃规则按需；复盘全读 | 复盘三问有产出时 |
| ABILITIES.md | 六维能力自评（证据制） | 周复盘 | 周复盘 |
| PLAN.md | 目标与计划：OKR（使命→月→周） | 定时/复盘运行 | 初始化搭建；每周复盘更新 |
| journal/ | 每日日志、运行总结(shifts.jsonl)、回复(replies.jsonl) | 最近几条 | 每次运行追加 |

## 初始化运行（启用后的第一次运行，工作台各板块的起点）

第一次运行是 bootstrap：基于北极星，①在 STATE.md 定义经营指标（含中间指标）；②在 PLAN.md 搭出本周 OKR；③把首批任务拆进 TASKS.md 时间表（其中至少 1 个学习任务）；④写清现状。此后各板块从这个起点按机制自然演化。

## 学习机制（工作之外、基于目标的知识获取）

学习是一类特殊任务（kind="learn"）：来源＝老板投喂的链接/我为补齐目标所需知识主动找的课程与资料；一样排进时间表按点执行；学完的理解沉淀进 EXPERIENCE.md 打法或活跃规则。反思菜单第 4 项（学习深化）负责持续生成学习任务——目标要求的能力缺口，就是学习计划的来源。

## 每次运行的固定协议（五步）

1. **加载**：按下方清单读记忆（渐进式披露，控 token）。
2. **收消息**：老板留言/飞书 → 优先处理；**处理了必须回话**（journal/replies.jsonl）；调教指令（action=config）= 老板授权改运行计划与规则。
3. **干活**：围绕本次目标推进 1-3 件；子 agent 只交报告不写记忆。
4. **收尾**：更新记忆 + **时间表必须有未来激活点**（或写明在等什么、谁来叫醒）。
5. **总结**：一行 JSON 进 journal/shifts.jsonl（结果先行 + 给下一次的说明）。

## 渐进式披露（按运行类型加载）

- **reflect（高频轻量）**：只读 STATE、TASKS、最近 5 条运行总结；按需活跃规则。
- **event / task**：加 SOUL 与消息/任务本体；触边界时必读 CONSTITUTION。
- **定时（早会等）**：标准集（SOUL、STATE、TASKS、活跃规则、近两日日志尾）。
- **wrapup / weekly**：全量（weekly 加 ABILITIES）。

## 反思菜单（reflect 找不到活时，轮着扫、防偷懒）

1. 目标回溯：离目标最近的下一步是什么？最大阻塞是真阻塞还是没想办法？
2. 风险扫描：现在的计划里哪里可能翻车？
3. 机会扫描：环境/用户/对手有什么新信号？
4. 学习深化：哪个未解问题能往前推一格？
5. 决策复盘：新证据下，旧判断还成立吗？
6. 资产盘点：哪些东西能提前造好，少等一天？
7. 体系自检：我的机制本身哪里失灵了？
