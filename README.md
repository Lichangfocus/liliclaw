# liliclaw — 在你电脑上 7×24 工作的数字员工

给自己雇一名（或一队）数字员工：选一个你本机的 agent CLI 当它的脑子，指一个 workspace，说清目标——它就开始像一个不知疲倦的人一样持续工作：自己定指标、拆 OKR、排时间表、按点干活、空闲自我反思找活、踩坑沉淀经验，并通过工作台和飞书随时向你汇报、听你指挥。

**框架自身零模型**：一切智能都来自你自选的内核 CLI（Claude Code / Kimi Code / pi / Qwen…），常驻部分是纯脚本，不外发任何数据；记忆全部是你本机的 Markdown 文件。

## 一分钟跑起来（macOS）

```bash
python3 server/server.py     # 打开 http://localhost:8866
```

页面上完成一切：新建数字员工 → 四步初始化（选内核：真实扫描本机已装的 CLI → 选 workspace：可以是你现有的项目仓库，记忆只写入其中的 `.liliclaw/` 子目录、绝不碰你的文件 → 基本设定：名字/角色/目标 → 初始化并启用：先对内核真跑一条连通测试，通过才上线）。启用后它的第一次运行会基于目标自动搭出经营指标、本周 OKR、首批任务与学习计划。

## 架构（三层，App 与 agent 生命周期解耦）

```
浏览器/桌面壳（只是显示器和遥控器，零本地能力）
    │ HTTP (localhost)
本地服务器 server/     ── 管理界面 + 工作台渲染 + 收件箱写入
常驻调度网关 core/bin/bell.py ── launchd 保活的纯脚本守护（系统的心脏）：
    │  工作台消息秒级强起运行 · 飞书 10s 轮询 · 时间表/定时/反思 30s 对表
运行执行器 core/bin/shift.py ── 每次被强起：渲染提示词(渐进披露) → 调起内核 CLI → 看门狗/痕迹检查
内核 CLI（用户自选，全部 AI 在此）
    │ 在 workspace 里干活，读写 .liliclaw/ 记忆
```

- **记忆 8 文件**：AGENT.md（生效组合契约）、SOUL（人格）、CONSTITUTION（规则）、STATE（目标/指标/现状）、PLAN（OKR）、TASKS（任务+时间表）、EXPERIENCE（经验）、ABILITIES（六维能力）+ journal/。
- **运行计划五件套**：定时（早会/复盘会）· 触发（你的消息秒级响应，永不打断主流程）· 反思（空闲自查找活）· 时间表（任务带激活时间）· 推进（每次运行必排下一步）。
- **保护**：单实例锁、20 分钟看门狗、失败退避+自动重试、初始化标记重试、留言机械移交、多员工 workspace 查重隔离。

## 多名员工

管理界面可创建多名数字员工：各自独立的内核、workspace、记忆、飞书机器人，一个网关统一调度、互不干扰（已实测双员工不同内核并行运行）。

## 路线

- [x] Phase 1a：核心框架 + 本地站点（当前，纯 Python + 零依赖前端）
- [ ] Phase 1b：Tauri 桌面壳（托盘、系统通知、开机自启、签名分发）——`app/`
- [ ] Phase 2：Windows（Task Scheduler 常驻 + 嵌入式 Python 打包；core 已无 bash，前端/协议零改动）

## 诚实边界

电脑关机/睡眠即停摆（建议常开或 Mac mini）；无头运行没有浏览器登录态/桌面控制，这类活会标注留给真人；飞书为轮询（本地无公网回调）；凭证只存本机文件并已全部 gitignore。
